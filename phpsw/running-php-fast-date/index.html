Running PHP fast

This month is all about monitoring, analysing and optimising the performance of your PHP apps! We'll talk about finding bottlenecks in your stack, tuning your Nginx and general performance hotspots in PHP applications.

- What is my code doing in the wild?
   Dave Avent, Site Reliability Engineer at SeatMe (Yelp)

How can I make my PHP application run faster in Production? Using System Administration techniques and modern DevOps tools you can find out where and why your code is not responding as it does in development.

- Faster Xdebug coverage analysis
    Warren Seymour, Software Developer at Radify

Code coverage analysis can be an incredibly useful metric for determining the overall quality of your test suite. However, as your codebase grows larger and more complex, so does the time taken to perform coverage analysis. I'll be sharing the tools and techniques we use at Radify to reduce our code coverage time by a factor of ten.

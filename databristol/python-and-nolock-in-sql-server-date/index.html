Python and NOLOCK in SQL Server

*** Please note that this is at a different Ovo office to the one we've held talks in previously (we'll be in Ovo Energy, 2nd Floor, 10 temple back, Bristol. BS1 6FL) ***

This month we have two of our Bristol members (James Yarrow and Niall Langley) doing talks... see below for a summary.

We hope you can join us for a couple of great talks and networking with free pizza and beer. These meetups attract a friendly, relaxed crowd. Look out for organisers with an orange lanyard if you need any info (or just for a chat!).

If you're a twitterer you can find us at https://twitter.com/SQLBristol and please do message us or comment below if you have any questions in advance.

James Yarrow - Applied Python in SQL Server 2017:

We are be looking at how we can use the feature to meet some common problems.

How we can extend stored procedures to talk to API's.
How we can create models for our customers.
How we can post data out to web hooks.

Python opens up a great selection of options and advantages.

Niall Langley - NOLOCK – The good, the bad & the ugly:

In this session we will look at the NOLOCK query hint, how and where it is commonly used, and use some demos to understand what problems this can cause. We will then look at the alternatives available, including a look at the various transaction isolation levels that we can use to help solve concurrency issues on our own servers.
